# 思源笔记 CalDAV 同步插件

将思源笔记属性表（Attribute View）与 CalDAV 日历进行双向同步，并提供可视化日历面板。



## 功能亮点

- **双向同步** — 思源 ↔ CalDAV 日历数据实时双向同步，增删改自动对齐
- **多账户支持** — 可同时接入多个 CalDAV 日历账户（个人/工作/家庭等）
- **日历面板** — 内置月/日/年三视图日历，直观浏览和管理事件
- **智能修复** — 自动检测并修复 UID 不匹配的孤儿事件
- **熔断保护** — 连续错误达到阈值自动暂停，防止数据误删
- **安全熔断** — 思源数据为空时拒绝同步，防止视图切换导致日历数据被清空
- **定时调度** — 可配置同步间隔，插件加载后自动运行
- **暗色适配** — 完整支持思源亮色/暗色主题

## 快速开始

### 第一步：创建思源属性表

在思源笔记中新建一个属性表（Attribute View），用于存储日历事件数据。


### 第二步：创建属性表列

按照下表创建 8 个列，并记录每列的 **列 ID**（点击列头 → 列设置 → 查看列 ID）：

| 列名 | 推荐类型 | 是否必填 | 说明 |
|------|---------|---------|------|
| 标题 | 文本（text） | ✅ | 事件标题 |
| 开始时间 | 日期（date） | ✅ | 事件开始时间 |
| 结束时间 | 日期（date） | ❌ | 事件结束时间 |
| UID | 文本（text） | ✅ | CalDAV 事件唯一标识（同步自动填充） |
| 同步时间 | 日期（date） | ✅ | 最后同步时间戳（同步自动更新） |
| 描述 | 文本（text） | ❌ | 事件描述内容 |
| 提醒 | 选择（select） | ❌ | 提醒规则，建议预设选项：无提醒、准点提醒、提前5分钟、提前15分钟、提前1小时、提前1天 |
| 账户 | 选择（select） | ✅ | CalDAV 账户名（同步自动填充） |


### 第三步：配置插件

1. 点击顶栏同步图标，或进入 `设置 → 插件 → CalDAV 同步 → 设置`
2. 填写 **数据库 ID**（属性表 URL 中的 ID，如 `2024xxxx`）
3. 填写各 **列 ID 映射**
4. 可选填写 **视图 ID**（锁定视图，防止切换视图导致同步误删）


### 第四步：添加 CalDAV 账户

1. 在设置面板的"CalDAV 账户"区域，点击 **+ 添加账户**
2. 填写账户信息：
   - **账户名称**：自定义名称，如"个人"、"工作"（需与属性表"账户"列的选项一致）
   - **服务器 URL**：CalDAV 日历端点地址
   - **用户名**：CalDAV 登录用户名
   - **密码**：CalDAV 登录密码
3. 点击 **测试连接** 确认可用


### 第五步：保存并启动同步

点击 **保存配置**，插件将自动启动定时同步。首次同步会全量比对所有事件。

## 功能详解

### 1. CalDAV 双向同步

#### 同步引擎原理

同步采用三阶段策略：

| 阶段 | 操作 | 说明 |
|------|------|------|
| 阶段零 | 智能修复 UID | 检测思源中 UID 不匹配的孤儿事件，通过标题+时间模糊匹配关联到 CalDAV 事件 |
| 阶段一 | 删除检测 | 对比缓存，检测任一侧已删除的事件，在另一侧执行同步删除 |
| 阶段二&三 | 变动扫描与内容同步 | 对比内容哈希，检测新增和修改的事件，按时间戳决定覆盖方向 |

#### 冲突策略

采用 **Last-Write-Wins**（最后写入胜出）策略：
- 比较思源 `同步时间` 与 CalDAV `X-SYNC-TIME` / `LAST-MODIFIED`
- 时间戳较新的一方覆盖较旧的一方
- 内容未变动（哈希一致）时跳过，避免无意义写入

#### UID 追踪与回退机制

- 每个事件通过 UID 唯一标识
- 若思源行缺少 UID 值，临时回退使用思源行 ID 作为 UID（`isFallbackUid` 标记）
- 下次同步时，阶段零会尝试通过标题+时间匹配找到正确的 CalDAV 事件并回写 UID
- 无法匹配的孤儿事件会生成新 UID 并推送到 CalDAV

#### 定时调度

- 插件加载后自动启动定时同步
- 同步间隔可在设置中配置（默认 30 分钟，最短 1 分钟）
- 支持手动触发"立即同步"

#### 熔断保护

- 连续同步错误次数达到阈值（默认 5 次）后自动暂停
- 暂停后可在同步状态面板中点击"解除熔断"恢复
- **安全熔断**：若思源数据为 0 但缓存有历史记录，拒绝同步（防止视图切换或 API 异常导致日历数据被误删）

#### 提醒同步

插件支持以下预设提醒选项的双向同步：

| 选项 | ICS Trigger |
|------|------------|
| 无提醒 | （无 VALARM 块） |
| 准点提醒 | `-PT0M` |
| 提前 5 分钟 | `-PT5M` |
| 提前 15 分钟 | `-PT15M` |
| 提前 1 小时 | `-PT1H` |
| 提前 1 天 | `-P1D` |

也支持自定义格式如"提前 30 分钟"、"之后 2 小时"等。

### 2. 日历面板

通过顶栏图标或同步状态面板的"打开日历"按钮进入。


#### 月视图

- **折叠状态**（默认）：只显示选中日期所在的一周行，事件用**彩色圆点**指示（每个圆点代表一个事件，颜色对应账户），最多显示 3 个圆点
- **展开状态**：显示完整 6×7 月网格，事件用**彩色条**展示（最多 2 条 + 更多提示）
- 底部展开/收起按钮，带旋转箭头动画
- 切换月份或选中日期变化时自动折叠
- 点击标题可切换到年视图


#### 日视图

- 全天事件以独立圆角卡片展示
- 每小时一行卡片（8px 圆角），当前小时渐变高亮 + 蓝色边框
- 事件行内显示账户颜色圆点标识
- 点击标题可切换到月视图


#### 年视图

- 3 列 × 4 行 12 月卡片网格
- 每个卡片包含：月标题、事件数蓝色徽章、星期行、迷你日历
- 有事件的日期加粗并显示蓝色圆点
- 当前月份蓝色边框 + 渐变高亮
- 点击某月卡片切换到月视图


#### 事件列表

- 显示选中日期的事件详情
- 每个事件前有账户颜色圆点标识
- 多账户事件一目了然


#### 自适应布局

月视图的事件区域根据屏幕和展开状态自动调整：

| 状态 | 事件位置 | 布局方向 |
|------|---------|---------|
| 大屏 + 折叠 | 月视图**下方** | 上下（column） |
| 大屏 + 展开 | 月视图**右侧** | 左右（row） |
| 小屏（≤640px） | 月视图**下方** | 始终上下 |

#### 交互操作

- **左右滑动**：切换上一月/下一月（移动端）
- **键盘左右方向键**：切换上一月/下一月（桌面端）
- **点击标题**：层级上钻（日 → 月 → 年）
- **"今天"按钮**：快速回到当前日期
- **ESC 键**：关闭日历面板

### 3. 设置面板


#### 基础配置

| 配置项 | 说明 | 默认值 |
|-------|------|-------|
| 数据库 (AV) ID | 属性表 ID，从属性表 URL 中获取 | — |
| 视图 ID（可选） | 锁定视图，防止切换视图导致同步误删 | — |
| 同步间隔（秒） | 定时同步的时间间隔 | 1800（30 分钟） |
| 最大连续错误 | 熔断保护的错误阈值 | 5 |

#### 列 ID 映射

填写属性表中各列的 ID，用于同步引擎读写数据。每列 ID 可在属性表的列设置中查看。

#### CalDAV 账户管理

- 支持添加多个 CalDAV 账户
- 每个账户独立配置：名称、服务器 URL、用户名、密码
- 内置连接测试功能，保存前可验证账户可用性

#### 操作按钮

| 按钮 | 说明 |
|------|------|
| 保存配置 | 保存当前配置并重启调度器 |
| 立即同步 | 手动触发一次完整同步 |
| 重置缓存 | 清空同步缓存，下次同步将重新全量比对 |

### 4. 同步状态面板


| 信息 | 说明 |
|------|------|
| 状态指示灯 | 🟢 空闲 / 🔵 同步中 / 🔴 错误 / ⚪ 已暂停 |
| 上次同步 | 最近一次同步完成时间 |
| 下次同步 | 下一次定时同步预计时间 |
| 上次结果 | 删除数 / 变动数 / 错误数 |
| 连续错误 | 当前连续错误计数 |
| 最近日志 | 最近 20 条同步日志（实时刷新，每 3 秒） |

| 操作 | 说明 |
|------|------|
| 立即同步 | 手动触发同步 |
| 解除熔断 | 重置错误计数，恢复定时同步 |

## CalDAV 服务器兼容性

### 服务器地址格式

服务器 URL 应指向 CalDAV 日历端点，而非账户根路径。例如：

| 服务器 | 示例 URL |
|--------|---------|
| Radicale | `https://caldav.example.com/dav/username/calendar/` |
| Nextcloud | `https://cloud.example.com/remote.php/dav/calendars/username/personal/` |
| Baikal | `https://dav.example.com/dav/calendars/username/default/` |
| iCloud | `https://caldav.icloud.com/dav/username/calendar/` |

> **注意**：URL 末尾一般需要 `/`，具体取决于服务器配置。

### 已知限制

- 插件使用思源的 `forwardProxy` API 发起 CalDAV 请求，需在思源桌面端或浏览器端运行
- `forwardProxy` 对非标准 HTTP 方法（PROPFIND / PUT / DELETE）的支持取决于思源版本
- 浏览器端直接访问时可能受 CORS 限制，建议在思源桌面端使用

## 常见问题（FAQ）

### 同步不生效？

按以下顺序检查：
1. 数据库 ID 是否正确？→ 在属性表 URL 中确认
2. 列 ID 映射是否完整？→ 至少需要"标题"、"开始时间"、"UID"、"同步时间"、"账户"五列
3. CalDAV 账户是否配置？→ 使用"测试连接"验证
4. 事件是否有开始时间？→ 缺少开始时间的事件会被跳过

### 事件被误删？

- 确认是否填写了 **视图 ID** — 切换属性表视图可能导致同步引擎读取到空数据，触发安全熔断
- 如果安全熔断触发，同步会自动中止，不会删除任何数据
- 检查同步状态面板的日志，查看具体删除原因

### 同步后 UID 列为空？

- 首次同步时，思源中手动创建的事件可能没有 UID
- 同步引擎会在阶段零自动生成 UID 并回写到 UID 列
- 如果回写失败，检查 UID 列的列 ID 是否正确

### 密码安全？

- 当前密码以明文存储在插件配置中（与原 `.env` 方式一致）
- 配置数据通过思源 `saveData/loadData` 存储，随思源数据目录同步
- 建议使用应用专用密码（如 Nextcloud 的 App Password）

### 如何重置同步？

1. 在设置面板点击"重置缓存"
2. 下次同步将重新全量比对所有事件
3. 不会删除已有数据，只会重新建立缓存映射

### forwardProxy 请求失败？

- 确保在思源桌面端运行（非纯浏览器环境）
- 检查 CalDAV 服务器是否可达
- 检查服务器 URL 格式是否正确（需包含日历路径）
- 部分服务器可能不支持 PROPFIND Depth:1 请求

## 开发指南

### 项目结构

```
src/
├── index.ts                   # 插件入口 CalDAVSyncPlugin extends Plugin
├── main.ts                    # init/destroy 生命周期 + 配置/缓存加载
├── App.vue                    # 根组件（自建 overlay/dialog + v-if 条件渲染）
├── index.scss                 # 全局暗色/亮色主题适配
├── env.d.ts                   # 类型声明
├── types/index.ts             # PluginConfig, SyncEvent, SyncCache 等类型定义
├── utils/
│   ├── uid.ts                 # generateUID（Web Crypto API）
│   ├── ics.ts                 # ICS 时间戳 ↔ 毫秒转换 + 文本转义
│   ├── reminder.ts            # 提醒规则解析（中文选项 ↔ ICS Trigger）
│   └── hash.ts                # getEventHash（djb2 哈希算法）
├── api/
│   ├── siyuan.ts              # 思源 AV 操作客户端（fetchSyncPost）
│   └── caldav.ts              # CalDAV 客户端（forwardProxy，PROPFIND/GET/PUT/DELETE）
├── sync/
│   ├── engine.ts              # 三阶段同步引擎核心
│   ├── cache.ts               # 缓存管理（saveData/loadData）
│   └── scheduler.ts           # 定时调度器（start/stop/runOnce/reset）
└── components/
    ├── SettingPanel.vue        # 设置面板
    ├── AccountConfig.vue       # CalDAV 账户配置 + 连接测试
    ├── SyncStatus.vue          # 同步状态面板
    ├── CalendarPanel.vue       # 日历面板容器（三视图 + 自适应布局）
    ├── MonthView.vue           # 月视图（折叠/展开 + 圆点/事件条）
    ├── DayView.vue             # 日视图（卡片式小时行）
    ├── YearView.vue            # 年视图（3×4 月卡片网格）
    └── EventList.vue           # 事件列表
```
